# AES-GCM 原生Python实现

本项目是基于Python原生语言对AES-GCM认证加密算法的全手工实现，旨在为密码学学习者提供可阅读、可调试的参考代码。项目严格遵循NIST规范，拆解了AES加密算法与GCM模式的核心逻辑，所有代码均经过随机测试验证与标准库比对，确保算法正确性。


## 项目简介

AES-GCM是一种同时提供机密性与完整性的认证加密算法，广泛应用于网络通信、数据存储等场景。本项目通过"拆解式实现"，将复杂的密码学算法拆分为可理解的模块，适合以下人群参考：
- 密码学入门者，希望理解AES与GCM的底层原理
- 开发者，需要对照代码学习认证加密算法的实现细节
- 学生，用于课程设计或密码学实验参考

----------------
**请注意： 本 GCM 模式**
- 支持可变长 IV（8 至 128 字节，对于长度小于 12 字节的 IV，须遵守 NIST 规范中提出的 “确定性构造”）
- 支持 128 bits 的分组密码（比如，AES）
- 支持 64 GB 以下的非空明文 / 密文的加密 / 解密，超过长度的需自行扩展多个 IV 进行加密 / 解密
- 支持 128 bits 的认证标签长度（即，Tag 长度），如需获取更短的认证标签，请自行截断标签（根据 最新的NIST 规范要求，标签长度不得小于 96 位，96，104 112，120，128 位均可）

## 项目结构

```
AES-GCM
├── .documentation        # 参考文档与规范
├── SBox                  # S盒与有限域运算模块
├── KeyExpansion          # 密钥扩展模块
├── BlockOperation        # 块加密运算模块
├── GCMmodules            # GCM模式模块
├── main.py               # 使用示例
└── utils.py              # 其他的一些工具函数
```


## 项目使用

1. **环境要求**：Python 3.6+（原生实现，无需额外依赖）
2. **基础使用**：直接运行各模块代码即可查看算法中间过程，例如：
   ```bash
   # 运行S盒生成测试
   python -m SBox.Box.py
   # 运行GCM完整功能测试
   python GCMmodules.GCM.py
   ```
3. **测试说明**：各个核心模块文件中均包含与Python标准库`cryptography`的对比测试，如需执行需安装：
   ```bash
   pip install cryptography
   ```
